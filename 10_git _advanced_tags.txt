2ï¸âƒ£ git tag -a <name> -m "message" â­ Annotated Tag (Recommended)

git tag -a v1.0 -m "Release 1.0"

What it does
  Creates an annotated tag
  Stores:
    Tagger name
    Date
    Message


View tag details
git show v1.0

Why annotated tags?
  Better for releases
  Signed possible (-s)
  Industry standard


Push annotated tag
  git push origin v1.0

Or push all tags:
  git push origin --tags

  Deletes local tag only

  git push origin --delete v1.0


Notes
  Local and remote tags are independent
  Deleting locally does NOT delete remote



Tag a Past Commit

Used when:
  Release decided later
  Tagging production commit
