A remote is a reference to another Git repository, usually:
  GitHub
  GitLab
  Bitbucket

What it does
  Adds a remote connection
  Gives it a short name (usually origin)

Why origin?
  Just a convention
  You can name it anything:
    git remote add upstream https://github.com/org/repo.git


Check remotes
git remote -v


Notes
  Does NOT push or pull
  Just saves the URL
  One-time setup



2Ô∏è‚É£ git remote -v

  Helps avoid pushing to wrong repo üò¨

What it does
  Sends local commits to the remote
  Updates remote branch


Notes
  Push only works if branch is up to date
  Fails if remote has newer commits




4Ô∏è‚É£ git pull <remote> <branch>

  Fetches changes
  Immediately merges them


‚û°Ô∏è Equivalent to:

Potential risk ‚ö†Ô∏è

Safer alternative (recommended)

Notes
  Use pull for quick sync
  Use fetch when cautious




5Ô∏è‚É£ git fetch <remote> ‚≠ê SAFEST
git fetch origin

What it does
  Downloads remote changes
  Does NOT touch your code
  No merge, no conflict

After fetch
git status
Your branch is behind 'origin/main'

Inspect changes:

git log origin/main
git diff main origin/main


Notes
  100% safe
  Preferred in professional teams
  Lets you review before merging
