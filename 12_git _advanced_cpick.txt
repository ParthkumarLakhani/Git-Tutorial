What is git bisect?
git bisect helps you find the exact commit that introduced a bug using binary search.
ğŸ‘‰ Instead of checking 100 commits one by one, Git finds it in ~7 steps.

How Bisect Works
  1.You tell Git:
      One good commit
      One bad commit
  2.Git checks the middle commit
  3.You test it and mark it good/bad
  4.Git repeats until it finds the culprit


Start Bisect
  git bisect start


Mark Bad (Current Commit)
  git bisect bad


Mark Good (Older Commit)
  git bisect good <commit-hash>

Example:
  git bisect good v1.0




Git Auto-checkout

Git will now:
  Checkout a commit
  Ask you to test

After testing:
  git bisect good   # if bug not present
  git bisect bad    # if bug present

Repeat until Git says:
  abc123 is the first bad commit
ğŸ¯ Bug found


End Bisect
  git bisect reset


Returns you to original branch.



Automation (Advanced / Production)
  git bisect run npm test


Git automatically:
  Runs tests
  Marks good/bad
  Finds the bug commit



When to Use Bisect

âœ… Bug introduced recently
âœ… Regression issues
âœ… Large commit history
